Mugs

Immutable collections library for JavaScript

Still very much under development so no source has been released yet.
If you're interested mail me at mahj@itu.dk

About

Mugs is an immutable collections library for JavaScript that consists of a powerful set of collection prototypes that will make it more natural to write JavaScript programs in a functional manner using features like higher-order functions. By functional programs I'm referring to programs that are referential transparent which it isn't possible to write if you only have mutable collections at your possession.

The library contains the following collections.

  • List a List contains a sequence of items in the order they were inserted.
  • Stack A stack inserts and retrieves items according to the last-in-first-out (LIFO) principle.
  • Queue A queue inserts and retrieves items according to the first-in-first-out (FIFO) principle.
  • TreeSet Set contains a sequence of items. The Set has the property that no item may appear twice.
  • LLRBSet a TreeSet based on a Left-Left Red Black Tree
  • HashSet a Set based on hashing
  • TreeMap a map stores items according to a given key.
  • LLRBMap a TreeMap based on a Left-Leaning Red Black Tree
  • HashMap a Map based on hashing
  • Multimap a Multimap is a map where each key is associated with a collection rather than a single item. The user is free to use any of the collections that implements the Extensible interface.
  • Random Access List a Random Access List is a List that supported fast retrieval of items stored at arbitrary positions in the List.

The collections have been developed with the following "interfaces" in mind.

  • Collection Is the base interface that all collections implements.
  • Extensible Is a collection with support for insertion and removal of items.
  • Indexed is a collection where the items can be accessed with an index
  • Sequenced is an collection whose items are stored in a sequence determined but an ordering of the items or the insertion order
  • Directed are for collections that have a sense of direction. This means they support operations like reverse, foldLeft and foldRight (amongst others) that require a sense of direction.
  • Map is a collection that associates a key with a single item.
  • Multimap is a collection that associates a key with a collection of items. The items can be stored in an Extensible collection specified by the user upon construction of the Multimap.
  • SortedMap is a Map where the keys are stored in a sorted order

The following resources have been used to implement the various data structures

  • Purely Functional Data Structures by Chris Okasaki's book has been used to implement the Red Black Tree and Queue
  • Left-leaning Red-Black Trees by Robert Sedgewick has been used to implement the Left-Leaning Red Black Tree
  • Purely Functional Random-Access List by Chris Okasaki has beed used to implement the Random Access List
